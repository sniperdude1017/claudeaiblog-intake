# Distribution Playbook

Use these channels to drive qualified traffic outside Google Ads. The goal is not generic AI exposure. The goal is people already using Claude who care about launch notes, Claude Code, Projects, Artifacts, and workflow changes.

## Best near-term channels

- `X / Twitter`
  - Post short source-backed takes on every Claude launch, Claude Code change, or Anthropic release note.
  - CTA angle: `Here is what changed, who it matters for, and what to test next.`
- `Reddit`
  - Focus on `r/ClaudeAI`, `r/Anthropic`, `r/LocalLLaMA`, `r/ChatGPTCoding`, and narrow workflow or coding subs only when the post is useful first.
  - CTA angle: `I summarized the latest Claude update in plain English with source links and practical takeaways.`
- `Hacker News`
  - Post only when there is a useful artifact or launch summary, not a generic signup pitch.
  - CTA angle: `Source-backed summary of the latest Claude model and Claude Code changes for builders.`
- `Discord / Slack communities`
  - Share one useful launch digest or workflow note, then link the landing page only where allowed.
  - CTA angle: `Weekly Claude release roundup with direct source links.`
- `LinkedIn`
  - Aim at teams and knowledge workers actively evaluating Claude in real workflows.
  - CTA angle: `What changed in Claude this week and what teams should actually care about.`

## Secondary channels

- `Indie Hackers`
- `Dev.to`
- `Medium`
- `Quora`
- `Newsletter swaps`
- `GitHub` if the Claude brief becomes a public tracker or open feed

## Post structure

Every post should follow this shape:

1. `What changed`
2. `Who it is for`
3. `What to test next`
4. `Direct source links`
5. `CTA to claudeaiblog.com/join.html`

Avoid posting as `sign up for my blog`. Post as a useful Claude update artifact first, then let the landing page be the follow-up step.
